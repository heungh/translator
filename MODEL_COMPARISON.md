# 번역 모델 비교 분석

성인소설 번역 시스템에서 사용하는 모델들의 특성과 차이점을 분석합니다.

## 테스트 환경

- **Track 1**: 일반 내용 번역 + 노골적 표현 태깅 (주요 LLM)
- **Track 2**: 노골적 표현 번역 (Cohere, Ollama, OpenRouter)
- **테스트 텍스트**: 한글 성인소설 샘플 (약 600자)

---

# Part 1: 주요 LLM 비교 (Track 1용)

## 1. Claude (Anthropic)

**사용 모델**: Claude 4.5 Sonnet, Claude 3.5 Sonnet v2, Claude 3.5 Haiku (Bedrock)

#### 장점
- **뛰어난 한국어 이해도**: 문맥과 뉘앙스를 정확히 파악
- **일관된 태깅**: [EXPLICIT] 태그를 정확하게 적용
- **문학적 품질**: 자연스럽고 세련된 영어 번역
- **지시 준수도 높음**: 프롬프트 지시사항을 잘 따름
- **AWS Bedrock 통합**: Cross-Region Inference 지원

#### 단점
- 노골적 표현 직접 번역 거부 (Track 2 필요)
- 비용이 상대적으로 높음

#### 추천도: ★★★★★ (Track 1 최적)

---

## 2. Amazon Nova

**사용 가능 모델**: Nova Pro, Nova Lite, Nova Micro (Bedrock)

#### 장점
- **AWS 네이티브**: Bedrock에서 직접 사용
- **비용 효율적**: Claude 대비 저렴
- **빠른 응답**: 특히 Lite/Micro 모델

#### 단점
- **한국어 능력 제한적**: Claude 대비 한국어 이해도 낮음
- **태깅 일관성 부족**: [EXPLICIT] 태그 적용이 불안정할 수 있음
- **문학적 품질**: Claude보다 기계적인 번역

#### 추천도: ★★★☆☆
> ⚠️ 한국어→영어 번역 품질 테스트 필요

---

## 3. ChatGPT (OpenAI)

**사용 가능 모델**: GPT-4o, GPT-4 Turbo

#### 장점
- **우수한 한국어 이해도**: Claude와 비슷한 수준
- **문학적 표현**: 자연스러운 영어 번역
- **넓은 컨텍스트**: 긴 문서 처리 가능

#### 단점
- **강한 콘텐츠 필터**: 성인 콘텐츠 번역 거부 가능성 높음
- **별도 API 키 필요**: AWS 통합 불가
- **비용**: API 사용료 발생

#### 추천도: ★★★☆☆ (일반 번역용)
> ⚠️ 성인 콘텐츠 태깅/번역에는 부적합할 수 있음

---

## 4. Gemini (Google)

**사용 가능 모델**: Gemini 2.0 Flash, Gemini 1.5 Pro

#### 장점
- **빠른 속도**: 특히 Flash 모델
- **한국어 지원**: 합리적인 수준의 한국어 이해
- **직접적인 번역**: 노골적 표현도 비교적 잘 처리

#### 단점
- **태깅 일관성**: Claude만큼 정확하지 않을 수 있음
- **때때로 과한 번역**: 원문보다 더 직접적
- **문맥 오류 가능**: 간혹 어색한 표현

#### 추천도: ★★★★☆
> Gemini 2.0 Flash는 Track 2에서도 활용 가능

# Part 2: Track 2 모델 비교 (노골적 표현 번역)

app.py 구현 순서: Cohere Command R → Ollama Dolphin → OpenRouter

---

## 1. Cohere Command R+ (Bedrock)

**Model ID**: `cohere.command-r-plus-v1:0`, `cohere.command-r-v1:0`

#### 장점
- **자연스러운 문체**: 문학적이고 읽기 편한 번역
- **일관된 품질**: 여러 번 테스트해도 안정적인 결과
- **적절한 수위**: 노골적이지만 과하지 않은 표현
- **한국어 이해도 높음**: 뉘앙스를 잘 살림
- **AWS 통합**: 별도 API 키 불필요, 비용 효율적

#### 단점
- 간혹 의역이 강한 경우 있음

#### 추천도: ★★★★★

> ⚠️ 참고: Cohere 단독으로 한국어→영어 전체 번역 시 품질 테스트 필요 (Track 1 없이 사용 시)

---

## 2. Ollama - Dolphin (Self-hosted)

**Model ID**: `dolphin-llama3:8b`, `dolphin-mixtral:8x7b` 등

#### 장점
- **로컬 실행**: 무료, 데이터 프라이버시 보장
- **Uncensored**: 콘텐츠 제한 없음
- **노골적 표현 잘 처리**: 번역 거부 없음
- **유연한 모델 선택**: 다양한 uncensored 모델 사용 가능

#### 단점
- **한국어 이해도 제한적**: 클라우드 모델보다 낮을 수 있음
- **하드웨어 요구**: 로컬 GPU/CPU 리소스 필요
- **설치 필요**: Ollama 설치 및 모델 다운로드

#### 추천도: ★★★★☆

> ✅ 테스트 결과: Dolphin 모델은 노골적 표현 번역을 잘 처리함
> ⚠️ 참고: Ollama 단독으로 한국어→영어 전체 번역 시 품질 테스트 필요

---

## 3. OpenRouter (Multiple Models)

다양한 모델을 API 하나로 접근 가능

### 3-1. Mistral Small 3.1 24B Instruct (기본값)

**Model ID**: `mistralai/mistral-small-3.1-24b-instruct`

#### 장점
- **균형 잡힌 성능**: 속도와 품질의 균형
- **합리적인 비용**: 대형 모델 대비 저렴

#### 단점
- **한국어 능력**: 대형 모델보다 제한적

#### 추천도: ★★★☆☆

### 3-2. Llama 3.3 70B Instruct

**Model ID**: `meta-llama/llama-3.3-70b-instruct`

#### 장점
- **문학적 표현**: 세련되고 묘사적인 번역
- **세부 묘사 풍부**: 감정과 동작을 상세히 표현

#### 단점
- **불필요한 프리앰블**: "I can provide a translation..." 같은 문구 포함됨
- **약간 완곡한 표현**: 직접적인 표현을 피하는 경향
- **불안정한 출력**: 때때로 번역 거부 가능성

#### 추천도: ★★★☆☆

### 3-3. Qwen 2.5 72B Instruct

**Model ID**: `qwen/qwen-2.5-72b-instruct`

#### 장점
- **직접적인 번역**: 원문에 충실한 번역
- **의성어 처리**: 자연스럽게 처리
- **깔끔한 출력**: 불필요한 설명 없음

#### 단점
- **약간 기계적**: 문학적 뉘앙스가 부족할 때 있음
- 간혹 완곡한 표현 사용

#### 추천도: ★★★★☆

### 3-4. Gemini 2.0 Flash

**Model ID**: `google/gemini-2.0-flash-001`

#### 장점
- **가장 직접적**: 노골적인 표현을 그대로 번역
- **빠른 속도**: Flash 모델답게 응답 빠름
- **대화체 살림**: 자연스럽게 처리
- **원문 충실도 높음**: 직역에 가까움

#### 단점
- **때때로 과한 표현**: 원문보다 더 직접적일 수 있음
- **문맥 오류 가능**: 간혹 어색한 표현

#### 추천도: ★★★★☆

> ⚠️ 참고: OpenRouter 단독으로 한국어→영어 전체 번역 시 품질 테스트 필요
